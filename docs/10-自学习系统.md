# 模块10：自学习系统

自学习系统是 Hermes Agent 的**持续改进引擎** — 在每次对话后自动审查并更新 Memory 和 Skills，无需用户指令。

## 一、架构概览

```
AIAgent.run_conversation() 完成一轮
    ↓
spawn_background_review_thread()
    ↓
后台守护线程
    ├── fork AIAgent（继承父 Agent 的运行时）
    ├── 注入对话快照
    ├── 运行审查提示词
    └── Agent 自主决定：
        ├── memory(action=add, ...)  → 保存用户偏好
        ├── skill_manage(action=patch, ...) → 修复过时 Skill
        ├── skill_manage(action=create, ...) → 创建新 Skill
        └── "Nothing to save." → 不做任何修改
    ↓
Curator（周期性维护）
    ├── 每 7 天 + 空闲 2 小时触发
    ├── 扫描 agent-created Skills
    ├── 自动归档 90 天未使用的 Skill
    └── 合并重叠的 Skills
```

## 三、Background Review（每轮审查）

### 3.1 触发时机

```python
# AIAgent.run_conversation() 结束后
# 在后台守护线程中异步运行
# 不阻塞主对话
spawn_background_review_thread(agent, messages_snapshot, prompt)
```

### 3.2 审查 Agent 的创建

```python
review_agent = AIAgent(
    model=agent.model,              # 继承父 Agent 的模型
    max_iterations=16,              # 限制迭代次数
    quiet_mode=True,                # 静默模式
    provider=agent.provider,        # 继承 Provider
    api_key=...,                    # 继承凭证
    skip_memory=True,               # 不触碰外部 Memory 插件
    parent_session_id=agent.session_id,  # 记录父会话
)
# 继承父 Agent 的缓存系统提示词 → 命中前缀缓存 → 节省 ~26% 成本
review_agent._cached_system_prompt = agent._cached_system_prompt
```

### 3.3 三种审查模式

| 模式 | 审查内容 | 提示词 |
|---|---|---|
| Memory Only | 用户画像信息 | `_MEMORY_REVIEW_PROMPT` |
| Skill Only | 技能库更新 | `_SKILL_REVIEW_PROMPT` |
| Combined | Memory + Skills | `_COMBINED_REVIEW_PROMPT` |

### 3.4 Memory 审查提示词（核心逻辑）

```
Review the conversation above and consider saving to memory if appropriate.

Focus on:
1. Has the user revealed things about themselves — their persona, desires,
   preferences, or personal details worth remembering?
2. Has the user expressed expectations about how you should behave, their work
   style, or ways they want you to operate?

If something stands out, save it using the memory tool.
If nothing is worth saving, just say 'Nothing to save.' and stop.
```

### 3.5 Skill 审查提示词（核心逻辑）

```
Review the conversation above and update the skill library.
Be ACTIVE — most sessions produce at least one skill update.

Signals to look for:
  • User corrected your style, tone, format, or approach
  • Non-trivial technique, fix, workaround emerged
  • A loaded skill turned out wrong or outdated

Preference order:
  1. UPDATE A CURRENTLY-LOADED SKILL
  2. UPDATE AN EXISTING UMBRELLA
  3. ADD A SUPPORT FILE under an existing umbrella
  4. CREATE A NEW CLASS-LEVEL UMBRELLA
```

### 3.6 不应该捕获的内容

```
Do NOT capture:
  • Environment-dependent failures（缺少二进制、未配置凭证）
  • Negative claims about tools（"browser tools do not work"）
    → 这些会固化为长期拒绝，即使问题已修复
  • Session-specific transient errors（重试后解决的错误）
  • One-off task narratives（"总结今天的市场"不需要 Skill）
```

### 3.7 安全隔离

```python
# 审查 Agent 的工具白名单
# 只能使用 memory 和 skill_manage 工具
# 其他工具在运行时被拒绝

# 危险命令自动拒绝
def _bg_review_auto_deny(command, description, **kwargs):
    return "deny"  # 永远拒绝，不调用 input()

# skip_memory=True → 不触碰外部 Memory 插件
# 继承父 Agent 的 _memory_store → 写入仍然到达 MEMORY.md/USER.md
```

## 四、Curator（周期性维护）

### 4.1 触发条件

```python
DEFAULT_INTERVAL_HOURS = 24 * 7   # 7 天运行一次
DEFAULT_MIN_IDLE_HOURS = 2        # 至少空闲 2 小时
DEFAULT_STALE_AFTER_DAYS = 30     # 30 天未使用 → 过时
DEFAULT_ARCHIVE_AFTER_DAYS = 90   # 90 天未使用 → 归档
```

### 4.2 状态持久化

```python
# ~/.hermes/skills/.curator_state
{
    "last_run_at": "2026-05-20T10:00:00Z",
    "last_run_duration_seconds": 45,
    "last_run_summary": "Archived 2 skills, patched 1",
    "paused": false,
    "run_count": 12
}
```

### 4.3 Curator 执行流程

```
maybe_run_curator()
    ├── 检查 paused 状态
    ├── 检查空闲时间 ≥ 2 小时
    ├── 检查距上次运行 ≥ 7 天
    ↓
spawn Curator Agent（辅助模型）
    ├── 扫描 agent-created Skills
    ├── 读取使用数据（skill_usage）
    ├── 分析 Skill 健康：
    │   ├── 使用频率
    │   ├── 最近使用时间
    │   ├── 代码质量
    │   └── 与其他 Skill 的重叠
    ↓
生成建议：
    ├── Pin    → 标记重要 Skill
    ├── Archive → 归档过时 Skill（可恢复）
    ├── Consolidate → 合并重叠 Skill
    └── Patch  → 修复问题
    ↓
执行建议（通过 skill_manage）
    ↓
保存 Curator 状态 + 备份
```

### 4.4 严格约束

```
✓ 只处理 agent-created Skills
✓ 不触碰用户手动创建的 Skills
✓ 不触碰 bundled/Hub-installed Skills
✓ 永不自动删除 — 只归档（可恢复）
✓ Pinned Skills 跳过所有自动操作
✓ 使用辅助模型（不污染主会话的提示缓存）
```

## 五、Skill 生命周期

```
[创建] → agent-created / user-created
    ↓
[活跃] → 定期使用
    ↓
[过时] → 30 天未使用 → Curator 标记
    ↓
[归档] → 90 天未使用 → Curator 自动归档
    ↓                          ↓
[恢复] ← 用户手动恢复      [永久删除] ← 仅用户手动

Pinned Skills 永远停留在 [活跃] 状态
```

## 七、数据流全景

```
AIAgent.run_conversation() 完成一轮
    ↓
spawn_background_review_thread()
    ├── 创建守护线程
    ├── 快照对话消息
    └── 选择审查模式（memory/skill/combined）
    ↓
_run_review_in_thread()
    ├── 创建 fork AIAgent
    │   ├── 继承父 Agent 的运行时（Provider、Model、凭证）
    │   ├── 继承缓存的系统提示词（前缀缓存命中）
    │   ├── skip_memory=True（不触碰外部 Memory 插件）
    │   └── 工具白名单（memory + skill_manage）
    ├── 注入对话快照 + 审查提示词
    ├── review_agent.run_conversation(review_prompt)
    │   ├── Agent 分析对话内容
    │   ├── 自主决定是否需要更新
    │   ├── 调用 memory(action=add, ...) → 保存用户偏好
    │   ├── 调用 skill_manage(action=patch, ...) → 修复 Skill
    │   └── 或返回 "Nothing to save."
    └── 通过 callback 通知用户审查结果
    ↓
[周期性] Curator
    ├── 空闲 + 间隔满足 → 触发
    ├── 扫描 agent-created Skills
    ├── 归档/合并/修复
    └── 保存状态
```

## 八、关键设计决策

| 设计决策 | 原因 |
|---|---|
| 后台线程（不阻塞主对话） | 用户体验不受影响 |
| 继承父 Agent 的缓存系统提示词 | 前缀缓存命中，节省 ~26% 成本 |
| skip_memory=True | 防止审查提示词泄露到外部 Memory 插件 |
| 工具白名单 | 审查 Agent 只能修改 Memory 和 Skills |
| 危险命令自动拒绝 | 审查 Agent 不调用 input()（防止死锁） |
| "Nothing to save" 是合法结果 | 避免为了更新而更新 |
| Curator 只归档不删除 | 归档可恢复，删除不可逆 |
| 不捕获环境相关错误 | 这些会固化为长期拒绝 |
| 不捕获负面的工具声明 | "X tool is broken" 会持续影响数月 |
