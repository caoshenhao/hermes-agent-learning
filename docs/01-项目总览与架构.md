# 模块1：项目总览与架构

Hermes Agent 是 Nous Research 开发的开源 AI Agent 框架，支持多平台接入（Telegram、Discord、Slack、微信、飞书等 20+ 平台）、多模型调用（OpenRouter、Anthropic、Google Gemini 等），是一个功能完整的通用型 AI 助手框架。

## 三、文件依赖链

Hermes 的模块依赖经过精心设计，避免循环导入。核心依赖链如下：

```
tools/registry.py          （零依赖 — 所有工具文件导入它）
       ↑
tools/*.py                 （每个文件在 import 时调用 registry.register()）
       ↑
model_tools.py             （导入 tools/registry + 触发工具发现）
       ↑
run_agent.py, cli.py       （顶层入口，导入 model_tools）
       ↑
gateway/                   （网关层，为每个消息创建 AIAgent 实例）
```

### 关键设计原则

- **tools/registry.py 零依赖**：所有工具文件导入它，而非反向依赖
- **工具自动发现**：通过 AST 分析 + importlib 动态导入，无需手动维护导入列表
- **惰性 OpenAI 导入**：`run_agent.py` 中 `from openai import OpenAI` 被延迟到首次使用（节省 ~240ms 导入时间）
- **初始化委托模式**：`AIAgent.__init__` 只是转发器，实际逻辑在 `agent/agent_init.py`

## 四、核心入口点

### 4.1 CLI 入口

```
cli.py → HermesCLI 类
  ├── 交互模式（默认）：持续对话
  ├── 单次模式（-c）：执行单条指令
  ├── TUI 模式（--tui）：启动 Ink 终端 UI
  └── 网关模式（gateway start）：启动消息网关服务
```

### 4.2 网关入口

```
gateway/run.py → start_gateway()
  ├── 加载配置 + 初始化适配器
  ├── 为每个平台创建消息监听器
  └── 每收到一条消息 → 创建 AIAgent 实例 → run_conversation()
```

### 4.3 直接调用

```python
from run_agent import AIAgent
agent = AIAgent(model="claude-sonnet-4", provider="openrouter")
result = agent.run_conversation("你好")
print(result["final_response"])
```

## 五、配置系统

### 5.1 配置文件层级

| 文件 | 用途 |
|---|---|
| `~/.hermes/config.yaml` | 主配置文件（模型、工具、平台设置） |
| `~/.hermes/.env` | 环境变量（API Key 等敏感信息） |
| `~/.hermes/profiles/<name>/` | 命名 Profile（独立配置、记忆、技能） |
| 项目根 `.env` | 项目级环境变量 |

### 5.2 Profile 系统

```python
# hermes_constants.py 核心函数
def get_hermes_home() -> Path:
    """返回 Hermes 主目录（profile 感知）
    
    优先级：
    1. ContextVar 覆盖（cron job 场景）
    2. HERMES_HOME 环境变量
    3. 平台默认路径（macOS/Linux: ~/.hermes, Windows: %LOCALAPPDATA%/hermes）
    """
```

### 5.3 路径解析

所有日志、状态数据库、技能文件、cron 任务都通过 `get_hermes_home()` 定位，确保 Profile 隔离。

## 六、日志系统

`hermes_logging.py` 提供三级日志文件（全部 Profile 感知）：

| 日志文件 | 级别 | 内容 |
|---|---|---|
| `~/.hermes/logs/agent.log` | INFO+ | 主运行日志 |
| `~/.hermes/logs/errors.log` | WARNING+ | 错误集中记录 |
| `~/.hermes/logs/gateway.log` | INFO+ | 网关运行日志 |

可通过 CLI 浏览：`hermes logs [--follow] [--level ...] [--session ...]`

## 七、状态管理

`hermes_state.py` 实现 `SessionDB`：

- **存储引擎**：SQLite，位于 `~/.hermes/sessions.db`
- **全文搜索**：FTS5 索引，支持 AND/OR/短语/通配符查询
- **会话持久化**：每条消息实时写入，崩溃恢复安全
- **跨会话检索**：`session_search` 工具基于此实现
